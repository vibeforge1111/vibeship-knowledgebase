## Content Outline

### Quick Answer (50 words max)
Learn vibe coding with this free 8-module course. Start by picking a tool (Cursor for most, Bolt for quick projects), then learn prompting, build projects, and integrate security practices throughout. No coding experience needed - just a computer and curiosity. Most learners ship their first app within a week.

### Course Overview Section
- What this course covers (8 modules)
- Who it's for (non-technical founders, designers, students, devs wanting to accelerate)
- Time commitment: ~10-20 hours total, self-paced
- Prerequisites: None (basic computer literacy)
- What you'll build: 3 real projects

### Module 1: What is Vibe Coding? (Foundations)
**Learning Objectives:**
- Understand what vibe coding is and isn't
- Know who coined the term and why it matters
- Understand the AI coding landscape
- Set realistic expectations

**Content:**
- Link to: [What is Vibe Coding?](/kb/vibe-coding/what-is-vibe-coding/)
- Brief history (Andrej Karpathy, Feb 2025)
- How vibe coding differs from traditional coding
- What you can realistically build
- Success stories and use cases

**Word count:** 150-200

### Module 2: Choosing Your First Tool
**Learning Objectives:**
- Understand the tool landscape
- Pick the right tool for your goals
- Set up your first tool

**Content:**
- Tool categories: IDEs (Cursor, Windsurf), Web-based (Bolt, Lovable, v0), CLI (Claude Code)
- Decision matrix based on user type:
  - Non-technical founder: Bolt or Lovable
  - Designer: v0 + Cursor
  - Developer accelerating: Cursor or Claude Code
  - Student learning: Bolt or Replit
- Beginner recommendation: Start with Bolt (free, web-based, instant)
- Link to: [Vibe Coding Tools Hub](/kb/vibe-coding-tools/)
- Links to individual tool guides

**Word count:** 200-300

### Module 3: Mastering AI Prompting
**Learning Objectives:**
- Write clear, effective prompts
- Iterate when results aren't right
- Understand prompt patterns

**Content:**
- The WHAT + HOW + SECURE + AVOID formula (from secure guide)
- Prompt examples for common tasks:
  - "Create a landing page"
  - "Build a user registration system"
  - "Add a database"
- Common prompt mistakes and fixes
- When to be specific vs. when to let AI decide
- Iterating: "That's not quite right, change X"

**Word count:** 300-400

### Module 4: Your First Project - Portfolio Site
**Learning Objectives:**
- Build a complete project from scratch
- Experience the full vibe coding workflow
- Deploy something live

**Content:**
- Step-by-step walkthrough using Bolt
- Starting prompt example
- Iterating on design
- Adding sections (About, Projects, Contact)
- Deploying to Netlify/Vercel
- Estimated time: 30-60 minutes

**Word count:** 300-400

### Module 5: Security Fundamentals for Vibe Coders
**Learning Objectives:**
- Understand why AI code has security issues
- Learn the 5 most common vulnerabilities
- Start thinking security-first

**Content:**
- WHY this matters: 45% of AI code fails security tests
- The "Five Security Sins" AI tools make:
  1. Hardcoded secrets
  2. Missing authentication
  3. Injection vulnerabilities
  4. Excessive permissions
  5. Client-side only validation
- Link to: [Secure Vibe Coding Guide](/kb/vibe-coding/secure-vibe-coding-guide/)
- Security prompting basics
- Setting up rules files

**Word count:** 300-400

### Module 6: Intermediate Project - Full-Stack App with Auth
**Learning Objectives:**
- Build a full-stack application
- Implement secure authentication
- Work with databases

**Content:**
- Project: Task manager with user accounts
- Tool: Cursor + Supabase
- Includes: Auth, database, CRUD operations
- Security practices applied:
  - Environment variables for secrets
  - RLS enabled on Supabase
  - Input validation
- Link to: [Next.js + Supabase Security](/kb/security/stacks/nextjs-supabase/)

**Word count:** 400-500

### Module 7: Advanced Techniques
**Learning Objectives:**
- Use multiple AI tools together
- Handle complex multi-file projects
- Debug effectively

**Content:**
- Claude Code for complex refactoring
- Using Cursor for daily coding + Claude Code for architecture
- Multi-file context management
- When to use Agent mode vs. Chat mode
- Handling errors and debugging with AI
- Link to: [Claude Code Guide](/kb/vibe-coding-tools/claude-code/)

**Word count:** 300-400

### Module 8: From Code to Production
**Learning Objectives:**
- Deploy securely
- Set up monitoring
- Know what to check before launch

**Content:**
- Pre-deployment security checklist
- Environment setup (dev vs. production)
- Deployment platforms: Vercel, Netlify, Railway
